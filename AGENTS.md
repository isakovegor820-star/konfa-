## 🥇 ГЛАВНЫЙ ПОЖИРАТЕЛЬ КОНТЕКСТА — КАРТИНКИ (скриншоты + Read .png/.jpg)

Замер двух реальных сессий (транскрипты 33 МБ и 16 МБ) показал: **90–97% объёма всех
результатов инструментов — это base64-картинки**, а НЕ тяжёлый `index.html`.
- 33 МБ-сессия: 86 `preview_screenshot` = 9.4 МБ (82% результатов) + 16 `Read` картинок.
- 16 МБ-сессия: 30 `Read` по `shots/*.png` = 3.2 МБ (90% результатов).
- Каждая картинка **остаётся в хвосте контекста и пере-отправляется на КАЖДОМ следующем
  шаге** → транскрипт раздувается в ~4.6× → агент тормозит и упирается в сжатие контекста.

Правила:
- 🧹 **Чисти контекст между задачами** (`/clear`). Это главный рычаг — убирает накопленные
  картинки, чтобы они не пере-оплачивались по 30–80 раз.
- 📸 **Минимум скриншотов.** Для проверки — 1 финальный кадр нужной секции, не серия
  промежуточных. Перед съёмкой ужми вьюпорт под секцию (`preview_resize`), чтобы кадр был легче.
- 🚫 **Не делай `Read` по `shots/*.png`, `live_shots/*.png`, `*.jpg`** — это выгрузки-артефакты.
  Смотри их через `open`, а не тяни base64 в модель.

## 🚫 ПРАВИЛО №2 (профилактика) — НЕ ЧИТАЙ index.html ЦЕЛИКОМ

`index.html` весит **~757 КБ**, и из них **673 КБ — это две строки** в районе **строк 215–225**:

| Строка | ~Размер | Что это |
|--------|--------|---------|
| `<script type="__bundler/manifest">` (стр. 216) | 423 КБ | бандл рантайма артефакта |
| `<script type="__bundler/template">` (стр. 224) | 250 КБ | экранированный HTML артефакта |

Полное чтение этого файла = **190–250 тыс. токенов** ≈ весь контекст за один `Read`. Именно из-за
этого агент тормозит. Поэтому:

- ❌ **НИКОГДА** не делай `Read(index.html)` без `offset`/`limit`. Не читай строки 215–225 — там нет
  ничего редактируемого, только сериализованные блобы.
- ✅ Чтобы найти нужный код — ищи **маркер аддона** `Grep -n "===== ИМЯ ADD-ON ====="` и читай только
  его диапазон (`Read` с `offset`+`limit`, обычно 50–170 строк).
- ✅ Нужна вся editable-зона? `Read(index.html, offset=227)` — это минует оба блоба и даёт все 15
  аддонов за ~24–29K токенов вместо ~195–236K на полный файл (экономия ~88%).
- ℹ️ В замеренных сессиях это правило УЖЕ соблюдалось (0 полных чтений) — держим его как защиту
  от регресса, но главная боль не здесь, а в картинках (см. секцию выше).
- ✅ Правки логики делай **в маленьком `*-addon.js`-файле** (он и есть источник правды по фиче),
  затем синхронизируй inline-копию.

## Архитектура: аддоны = двойник .js ↔ inline в index.html

Каждая фича — это самодостаточный IIFE-аддон, который существует **в двух местах** и
**синхронизируется руками**:

1. отдельный файл `legal-ai-os-serve/<name>-addon.js` (удобно читать/редактировать);
2. inline-копия в `index.html` между маркерами `<!-- ===== <NAME> ADD-ON ===== -->` + `<script>…</script>`.

При правке аддона меняй **оба** места одинаково. При добавлении нового — создай `.js` и вставь такой же
блок перед концом `<body>`, по образцу соседних.

Аддоны переживают пересборку dc-runtime через комбо: гард `window.__ptX`, инжект CSS по `id`,
`MutationObserver` + короткий `setInterval`-поллинг + `load`/`DOMContentLoaded`. Поиск целевого узла —
по **тексту** (`querySelectorAll` + regex по содержимому), т.к. классы артефакта нестабильны.

⚠️ **Ищешь узел по тексту?** ОБЯЗАТЕЛЬНО ищи только внутри `<body>` и исключай `<script>/<style>/<template>`.
Иначе матчишь блоб `<script type="__bundler/template">` (стр. 224 — в нём весь HTML артефакта как текст,
со всеми заголовками страницы), и `innerHTML`-замена этого узла/его родителя **сносит бандлер-теги** →
сборка падает с `[bundler] Missing script tags`, приложение не монтируется. Доп. страховка: не трогай узлы,
у которых внутри есть `script[type^="__bundler"]`. (Образец безопасного поиска: `courses-addon.js` → `findGrid`.)

⚠️ **Заменяешь `innerHTML` контейнера, найденного как LCA по тексту?** После рендера твоя разметка
содержит тот же текст-якорь → LCA-поиск найдёт уже ТВОЙ узел и отрисует разметку внутри разметки →
бесконечная вложенность (`grid>grid>…`, всё схлопывается). Страховка: при подъёме возвращай `null`,
если встретил свой маркер-класс (образец: `toc-addon.js` → `findList`).

⚠️ **Делаешь `.glass`-карточку «без квадрата»?** Гаси рамку/тень/подъём не только в покое, но и на
`:hover`/`:focus`/`:focus-within`. У артефакта есть `.glass:hover { border-color/box-shadow/transform !important }`
— иначе коробка возвращается при наведении/клике. Бей бóльшей специфичностью:
`.glass.pt-XX-card:hover{ transform:none!important;border-color:transparent!important;box-shadow:none!important }`
(образцы: `pt-ac-card`, `pt-pr-bare`).

Аддоны в index.html (ищи по маркеру, не по номеру строки — они дрейфуют):
